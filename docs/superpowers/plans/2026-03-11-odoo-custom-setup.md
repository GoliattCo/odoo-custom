# Odoo 18.0 Custom Setup Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create a local Dockerized Odoo 18.0 environment with jorels-odoo-addons, pushed to a private GitHub repo at `remcaro-rgb/odoo-custom`.

**Architecture:** Odoo 18.0 source is included as a read-only git submodule for full reproducibility. jorels-odoo-addons is a second submodule from GitLab. Docker Compose runs PostgreSQL 15 + Odoo built from source via a custom Dockerfile.

**Tech Stack:** Python 3.12-slim (Debian Bookworm), PostgreSQL 15-alpine, Docker Compose, wkhtmltopdf 0.12.6.1-3 (patched-Qt from wkhtmltopdf/packaging), GitHub CLI (`gh`), Git submodules.

## Chunk 2: Docker Configuration

### Task 3: Write Dockerfile

**Files:**
- Create: `/Users/manuelcaro/Odoo/Dockerfile`

- [ ] **Step 1: Write Dockerfile**

Create `/Users/manuelcaro/Odoo/Dockerfile`:

```dockerfile
FROM python:3.12-slim

LABEL maintainer="Manuel Caro"

# System dependencies — all in one RUN layer to avoid stale cache
RUN apt-get update && apt-get install -y --no-install-recommends \
    build-essential \
    libpq-dev \
    libxml2-dev \
    libxslt1-dev \
    libldap2-dev \
    libsasl2-dev \
    libjpeg-dev \
    libssl-dev \
    libffi-dev \
    curl \
    gnupg \
    ca-certificates \
    fonts-noto-cjk \
    # wkhtmltopdf runtime deps
    fontconfig \
    libfreetype6 \
    libjpeg62-turbo \
    libpng16-16 \
    libx11-6 \
    libxcb1 \
    libxext6 \
    libxrender1 \
    xfonts-75dpi \
    xfonts-base \
    # Node.js (Bookworm ships Node 18)
    nodejs \
    npm \
    && rm -rf /var/lib/apt/lists/*

# wkhtmltopdf — patched-Qt build from wkhtmltopdf/packaging (NOT distro package)
# Bookworm amd64 native package — same source as official odoo:18 Docker image
# apt-get update required again here because lists were cleared in the prior layer
ARG WKHTMLTOPDF_VERSION=0.12.6.1-3
RUN apt-get update \
    && curl -sSL -o /tmp/wkhtmltox.deb \
    "https://github.com/wkhtmltopdf/packaging/releases/download/${WKHTMLTOPDF_VERSION}/wkhtmltox_${WKHTMLTOPDF_VERSION}.bookworm_amd64.deb" \
    && apt-get install -y --no-install-recommends /tmp/wkhtmltox.deb \
    && rm /tmp/wkhtmltox.deb \
    && rm -rf /var/lib/apt/lists/*

# less CSS compiler (node-less apt package not available on Bookworm)
RUN npm install -g less

# Python dependencies from Odoo source
COPY odoo/requirements.txt /tmp/requirements.txt
RUN pip install --no-cache-dir -r /tmp/requirements.txt

# Install Odoo itself (makes odoo-bin available as a Python package)
COPY odoo /odoo
RUN pip install --no-cache-dir -e /odoo

# Odoo filestore directory
RUN mkdir -p /var/lib/odoo

EXPOSE 8069

# Run as non-root user for security
# --no-create-home because /odoo already exists (populated by COPY above)
RUN useradd --no-create-home -u 1000 odoo \
    && chown -R odoo:odoo /var/lib/odoo
USER odoo

CMD ["/odoo/odoo-bin", "--config=/etc/odoo/odoo.conf"]
```

- [ ] **Step 2: Write docker-compose.yml**

Create `/Users/manuelcaro/Odoo/docker-compose.yml`:

```yaml
services:
  db:
    image: postgres:15-alpine
    env_file: .env
    environment:
      - POSTGRES_USER=${POSTGRES_USER}
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
      - POSTGRES_DB=${POSTGRES_DB}
    volumes:
      - odoo-db-data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER} -h localhost"]
      interval: 5s
      timeout: 5s
      retries: 10
    restart: unless-stopped

  odoo:
    build: .
    depends_on:
      db:
        condition: service_healthy
    env_file: .env
    volumes:
      - ./odoo:/odoo
      - ./addons/jorels-odoo-addons:/mnt/jorels-addons
      - ./custom-addons:/mnt/custom-addons
      - ./config/odoo.conf:/etc/odoo/odoo.conf:ro
      - odoo-filestore:/var/lib/odoo
    ports:
      - "8069:8069"
    restart: unless-stopped

volumes:
  odoo-db-data:
  odoo-filestore:
```

## Chunk 3: Build, Verify & Push

### Task 6: Build Docker image

- [ ] **Step 1: Verify submodules are initialized before building**

The Dockerfile copies `odoo/requirements.txt` and `odoo/` — if the submodule is not initialized, these paths will be empty and the build will fail silently.

```bash
git submodule status
```

Both lines must start with a commit SHA (no `-` prefix). If you see `-`, run:
```bash
git submodule update --init --recursive
```

- [ ] **Step 2: Build the Docker image**

```bash
cd /Users/manuelcaro/Odoo
docker compose build
```

Expected: build completes with no errors. This takes 5–15 minutes on first run (downloads base image, installs packages, installs Python deps from Odoo's requirements.txt).

**Troubleshooting:**
- `apt-get` errors → check internet connection; retry
- `wkhtmltopdf` download fails → wkhtmltopdf version is `0.12.6.1-3`, URL is in the `WKHTMLTOPDF_VERSION` ARG in the Dockerfile. Verify: `curl -I https://github.com/wkhtmltopdf/packaging/releases/download/0.12.6.1-3/wkhtmltox_0.12.6.1-3.bookworm_amd64.deb`
- Python package errors → verify `odoo/requirements.txt` exists: `ls odoo/requirements.txt`
- Any error about missing files → submodule not initialized (see Step 1)

- [ ] **Step 3: Verify image was created**

```bash
docker compose config --images
```

Expected: shows the image name for the `odoo` service (e.g., `odoo-odoo` or similar based on your directory name).

### Task 7: Run and verify Odoo

- [ ] **Step 1: Start services**

```bash

```

Expected: both `db` and `odoo` containers start.

- [ ] **Step 2: Check db healthcheck passes**

```bash
docker compose ps
```

Wait up to 60 seconds for `db` to show `healthy`. On a cold start with a fresh data volume, postgres initialization takes 30–90 seconds. Re-run `docker compose ps` if it still shows `starting`.

- `healthy` = postgres is accepting connections
- `starting` = still initializing, wait and retry
- `unhealthy` = check logs: `docker compose logs db`

Expected final state: `db` = `healthy`, `odoo` = `running`.

- [ ] **Step 3: Watch Odoo startup logs**

```bash

```

Wait for the HTTP server ready line (most reliable signal):
```

```
Or alternatively: `odoo.modules.loading: Modules loaded.`

Press Ctrl+C to stop following logs.

If Odoo fails to connect to DB: check `db` is healthy (`docker compose ps`), then: `docker compose restart odoo`.
