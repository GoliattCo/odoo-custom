Image-Digest Autocomplete QoL

Replace the operator-mint form's free-text 64-hex image_sha256 field with a combobox autocompleting recent enterprise-v* tags from GHCR. Reduces transcription error on a 64-character hash; reduces operator time per mint.

πHow the dropdown gets its data

tRPC enterpriseLicenses.recentEnterpriseImages()

  1. Operator clicks the combobox on /licenses/new.
  2. Client tRPC call hits the operator-gated procedure.
  3. Server checks 60s in-memory cache. Cache miss → continue; hit → return cached list.
  4. Server calls GET https://api.github.com/users/<OWNER>/packages/container/odoo-saas-odoo-enterprise/versions with optional GITHUB_PACKAGES_TOKEN (falls back to anonymous for public packages).
  5. Filter results to versions with at least one tag matching ^enterprise-.
  6. Map to {tag, digest, createdAt, sizeBytes}; sort desc by createdAt; limit 10.
  7. Cache + return.
  8. Combobox renders the dropdown.

⚠Failure modes — graceful degrade

GHCR API 404

Package doesn't exist (yet)

Procedure returns empty list. Combobox renders "No recent images" — operator types manually.

GHCR API 403

Rate-limited (60 req/hr anon)

Logs warn, returns empty. With 60s cache, this is hard to hit; with token, 5000 req/hr ceiling.

Network down

fetch throws

Caught; empty list returned. Operator types manually. No UI broken state.

GitHub returns garbage

Schema parse fail

Zod-validate the response; bad entries dropped from the list. Other valid entries still surface.

TTest coverage

Unit tests cover the procedure end-to-end with mocked GHCR responses. No integration test required — pure HTTP wrapping with no DB.

?Open questions

Public vs private GHCR package?
Today odoo-saas-odoo-enterprise is GHCR-private. Making it public is the right call. If still private when this lands, set GITHUB_PACKAGES_TOKEN in Vercel admin env (read:packages scope).
Cross-org packages?
Owner could change (remcaro-rgb → goliatt-co org). Read GHCR_OWNER from env, fall back to GITHUB_REPOSITORY_OWNER.
Cache strategy at scale?
In-memory Map cache is per-function-instance. With Vercel Fluid Compute reusing instances, hit rate should be fine. Move to Runtime Cache if it becomes a problem.
Tag filter strictness?
Include any tag matching ^enterprise-, not just ^enterprise-v*. Runbook allows enterprise-<customer-slug> tags for customer-specific builds.
